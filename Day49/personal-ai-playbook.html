Welcome to your Personal AI Playbook

Reusable systems, not stored prompts

Most people collect prompts. This playbook teaches the parts a good prompt is made of, so you can assemble thousands of variations from a small library of blocks. Everything you save lives on your machine (localStorage) — nothing leaves this file.

A prompt is not a spell. It's a stack of these parts: ROLE who the model should act as OBJECTIVE what you want CONTEXT what the model doesn't know CONSTRAINTS what's off-limits REASONING how it should think FORMAT the shape of the output TONE the voice EXAMPLES what "good" looks like QUALITY self-checks before finishing

The Prompt Builder

Click a block to add it. Each block has a dropdown of preset options plus a free-text override. Every block explains what it does and why it's there — you never have to guess what "Reasoning strategy" means. Live preview updates as you assemble.

Example: "Rewrite my resume bullet" ROLE ATS-optimization recruiter OBJECTIVE rewrite the bullet for keyword match CONTEXT target JD pasted below CONSTRAINTS under 25 words, no buzzwords FORMAT single bullet, no preamble QUALITY flag any invented achievements → Save. Reuse the same 6 blocks for every bullet.

The Loop Builder

Any prompt can become a self-improving loop. Paste your base prompt, define a goal, an evaluator, and stop conditions. The output is a single prompt that instructs the model to run itself in a loop until it hits a quality threshold, plateaus, or reaches a hard iteration cap.

Wrap a resume-bullet prompt into a loop: GOAL maximize quality score 1-100 EVAL clarity + specificity + JD match IMPROVE patch only the weakest point STOP score ≥ 90 OR no gain 2× OR cap 6 SAFETY abort if bullet invents a metric → Model iterates itself. You paste the winner.
