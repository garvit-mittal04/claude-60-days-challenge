📚 How this was researched · methodology + conflicts resolved

▼

Data sources. Pricing pulled from each vendor's published API pricing page (linked in the Sources panel). Capability scores drawn from official model cards + widely-cited public benchmarks (MMLU, SWE-bench Verified). Context windows and rate limits from official documentation.

Blended cost estimation. "Blended $/M tokens" assumes a 2:1 input-to-output token ratio (typical for chat and coding workloads). Actual cost varies by workload — heavy input-context work (long documents) skews cheaper; heavy generation (writing) skews closer to the output rate. Marked EST throughout.

Openness / Deployment Flexibility scoring. Judgment score 1-10 based on: proprietary-only vs. open weights (Llama), fine-tuning availability, self-hosting options. Not sourced from a single benchmark. Marked JUDGMENT.

Conflicts resolved.

  • Llama pricing varies wildly by host. Together.ai, Groq, Fireworks, and Replicate each price Llama differently. I used Together.ai's published price for the 70B tier as a representative reference point. Groq is typically cheaper for latency-optimized workloads; Fireworks is often mid-price with better throughput. Marked EST.
  • Context window vs. context quality. Both GPT-4.1 and Gemini 2.5 Pro advertise 1M-token context windows, but performance degrades past ~200K in independent testing. The score uses advertised context; a "quality-adjusted context" score would give both a lower rating.
  • SWE-bench versions. SWE-bench Verified (a curated 500-problem subset with human-verified solutions) is the current standard. Older SWE-bench scores from the original 2,294-problem set are not directly comparable. All numbers used are from Verified.
  • MMLU is saturating. Every frontier model now scores 85%+, so MMLU no longer meaningfully separates leaders. Included for completeness; treat 2-3 point differences as noise.

Update cadence. Numbers in this space change every 30-60 days. Rerun the comparison quarterly.
